# 微型商店 - 用户体验改进总结

## 📋 已完成改进项目

### 1. ✅ 分离用户注册页面
**问题**: 普通用户和商家用户没有区分，注册时会产生混淆

**解决方案**:
- 创建 `/register` 页面作为选择入口 - 展示两种用户类型的卡片界面
- 创建 `/register/user` 页面专供普通用户注册
- 创建 `/register/merchant` 页面专供商家用户注册
- 每个页面都有明确的说明和视觉区分（不同颜色和图标）

**新文件**:
- `templates/register.html` - 选择页面（蓝色和绿色卡片）
- `templates/register_user.html` - 普通用户注册（蓝色主题）
- `templates/register_merchant.html` - 商家注册（绿色主题）

---

### 2. ✅ 修复导航栏权限
**问题**: 普通用户的导航栏中也出现"商家中心"按钮，意味着普通用户也能上架商品

**解决方案**:
- 在 `base.html` 导航栏中添加权限检查：
  ```html
  {% if current_user.is_merchant or current_user.is_admin %}
    <li class="nav-item"><a class="nav-link" href="{{ url_for('merchant_dashboard') }}">...</a></li>
  {% endif %}
  ```
- 现在只有商家和管理员能看到"商家中心"选项

---

### 3. ✅ 用户类型颜色区分
**问题**: 普通用户、商家、管理员在个人中心没有视觉区分

**解决方案**:
- **导航栏**: 用户名旁添加彩色徽章
  - 👑 **管理员** - 红色徽章 (badge bg-danger)
  - 🏪 **商家** - 绿色徽章 (badge bg-success)
  - 👤 **普通会员** - 青色徽章 (badge bg-info)

- **个人页面**: 头像背景使用渐变色区分
  - 管理员: 红色渐变 + 盾牌图标
  - 商家: 绿色渐变 + 店铺图标
  - 普通用户: 青色渐变 + 用户图标

---

### 4. ✅ 搜索功能
**问题**: 用户无法搜索想要的商品

**解决方案**:
- 在导航栏添加搜索栏（紧靠品牌名称右侧）
- 支持按商品名称和描述搜索（不区分大小写）
- 搜索结果显示匹配的商品和搜索关键词提示
- 后端实现模糊搜索：`ilike` (SQL LIKE 查询)
- 记录搜索日志：`app.logger.info(f'用户搜索: {query}, 找到 {len(products)} 个商品')`

**搜索语法**:
- URL: `/?q=搜索关键词`
- 搜索字段: Product.name 和 Product.description
- 示例: `/?q=手机` 会找到所有名称或描述包含"手机"的商品

---

### 5. ✅ 首页排版重新设计
**问题**: 所有商品平铺展示，无法有效分类浏览

**解决方案**:
- 按商品分类组织显示
  - 每个分类单独成块
  - 显示分类名称和商品数量标签
  - 分类间用分割线区隔

- 未分类商品单独成块
  - 如果有商品未指定分类，单独显示在"其他商品"区域

- 搜索结果专门显示
  - 搜索时直接显示符合条件的商品
  - 显示搜索关键词和找到的商品数量

- 改进的卡片设计
  - 增加阴影效果 (shadow-sm)
  - 悬停时动画效果 (hover-shadow)
  - 更紧凑的商品信息布局
  - 4列响应式网格（大屏）→ 3列（中屏）→ 1列（小屏）

**示例布局**:
```
┌─────────────────────────────────────────┐
│ 📱 电子产品 (5件商品)                   │

┌─────────────────────────────────────────┐
│ 👗 服装配饰 (8件商品)                   │
├─────────────────────────────────────────┤
│ [商品卡片1] [商品卡片2] [商品卡片3] ... │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
```

### 6. ✅ 日志系统
**问题**: 没有集中的日志管理，不便调试和监控

**解决方案**:
- 创建专门的 `logs/` 文件夹
- 配置 RotatingFileHandler 实现日志轮转
  - 最大文件大小: 10MB
  - 保留最近10个备份文件
  - 文件名: `logs/app.log`

- 记录的事件:
  - ✅ 应用启动: `微型商店應用啟動`
  - ✅ 用户登入: `{用户类型} {用户名} 已登入`
  - ✅ 用户登出: `用户 {用户名} 已登出`
  - ✅ 用户注册: `新{用户类型}用户註冊: {用户名}`
  - ✅ 登入失败: `登入失敗: 用户名 {用户名}`
  - ✅ 产品搜索: `用户搜索: {查询词}, 找到 {数量} 个商品`

- 日志格式:
  ```
  2026-01-09 15:58:27,547 INFO: 微型商店應用啟動 [in /workspaces/FYP-A20/app.py:29]
  ```
